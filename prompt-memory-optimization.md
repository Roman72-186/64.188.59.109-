# Промт: оптимизация долгосрочной памяти проекта под минимум токенов

> Переиспользуемый промт-инструкция для другого агента (Claude Code). Проектно-нейтральный —
> подойдёт любому действующему проекту. Скопировать целиком блок ниже и выдать агенту как задачу.

---

```markdown
# ЗАДАЧА: оптимизировать долгосрочную память проекта под минимум токенов

Ты работаешь в УЖЕ ДЕЙСТВУЮЩЕМ проекте. Нужно настроить долгосрочную память
с минимальной тратой токенов и на сохранение, и на чтение. Действуй строго по
правилам ниже. Ничего не ломай и ничего не теряй.

## ПРИНЦИП (почему так)
Токены тратятся в двух местах:
- ЧТЕНИЕ — на то, что грузится в контекст КАЖДУЮ сессию (CLAUDE.md, индекс памяти).
- ЗАПИСЬ — на то, что генерится при сохранении факта.

Самая дешёвая долгосрочная память — ФАЙЛОВАЯ, двухуровневая:
  1) тонкий «индекс», который грузится всегда (одна короткая строка = один указатель);
  2) «тело» детали, которое читается ТОЛЬКО когда задача его касается (on-demand).

ЗАПРЕЩЕНО для session-памяти: memoir, векторные БД, RAG, любые системы, которые
гоняют LLM/эмбеддинги при сохранении ИЛИ поиске — это LLM-налог на обоих концах,
ровно то, чего мы избегаем. Если такое уже установлено только ради памяти агента —
предложи удалить.

## ШАГ 1. Аудит текущего состояния (только чтение)
- Прочитай корневой CLAUDE.md (и пользовательский, если есть). Посчитай строки/размер.
- Найди, что ещё авто-грузится каждую сессию (индексы памяти, большие doc-файлы).
- Определи, что в CLAUDE.md — это «ядро» (нужно почти всегда / safety-critical),
  а что — справочник (нужен изредка под конкретную задачу).
- НЕ начинай править, пока не составил список «оставить в ядре / вынести».

## ШАГ 2. Раздели CLAUDE.md на ядро + on-demand docs

В ЯДРЕ (грузится всегда) оставить ТОЛЬКО:
- Quick commands (тест/запуск/деплой/миграции) — используются постоянно.
- Working style / методология работы и фикса багов.
- КРИТИЧНЫЕ правила инфраструктуры (НЕ нарушать): деплой, прод-топология,
  правила, нарушение которых роняет прод/старт. Их прятать НЕЛЬЗЯ.
- Карту «Где что искать» — таблицу-указатель на docs-файлы.
- Краткий блок про память и сессии.

ВЫНЕСТИ в `docs/*.md` (читаются по требованию):
- Детальную архитектуру (lifecycle, auth, RBAC, модели данных, сервисы, интеграции, тесты).
- Полную структуру папок воркспейса.
- Справочники по подсистемам (workflows, интеграции и т.п.).
- Списки агентов/команд/skills, процедуры.
- Историю ручных prod-операций / точки отката.

Каждый docs-файл начинай строкой-маркером:
«> On-demand справочник. Читать, когда задача касается <темы>.»

## ШАГ 3. Карта в ядре
Добавь в CLAUDE.md таблицу вида:
| Тема | Файл |
|------|------|
| Архитектура … | docs/architecture.md |
| … | … |
И фразу: «Читать соответствующий файл, когда задача его касается».

## ШАГ 4. Файловая память (если её нет — создать)
- Один факт = один маленький .md-файл (короткое тело, без «простыней»).
- Индекс MEMORY.md = одна строка на факт (заголовок + хук-однострочник).
- В контекст грузится ТОЛЬКО индекс; тела читаются выборочно.
- Запись факта = прямой текст, без LLM-суммирования.
- Перед сохранением проверяй, нет ли уже файла на эту тему — обновляй, не дублируй.
- Не сохраняй то, что и так выводится из кода/гита/CLAUDE.md.

## ЖЁСТКИЕ ОГРАНИЧЕНИЯ
- Контент НЕ удалять — только ПЕРЕМЕЩАТЬ 1:1. Перенос ≠ редактирование смысла.
- Safety-critical правила (деплой, миграции, прод) обязаны остаться в ядре.
- Сохрани все факты, даты, id, точки отката дословно.
- Сначала покажи план разреза (что в ядро / что в docs) и дождись «ок»,
  потом правь файлы.

## ПРОВЕРКА (в конце)
- Покажи строки/размер: было CLAUDE.md → стало (ядро) + список docs с размерами.
- Цель: ядро меньше в ~2–3 раза, при этом 0 потерь контента.
- Подтверди, что каждый вынесенный кусок имеет ссылку в карте «Где что искать».
```

---

## Заметки по использованию
- Промт проектно-нейтральный — не привязан к конкретному коду.
- Ключевая защита от поломки — пункты «контент не удалять, только перемещать 1:1» и
  «safety-critical остаётся в ядре».
- Имена docs-файлов агент подбирает сам под свой проект; структуру задаёт карта «Где что искать».
- Эталон применения этого промта — данный воркспейс (`CLAUDE.md` + `docs/architecture.md`,
  `workspace-structure.md`, `n8n-workflows.md`, `agents-and-skills.md`, `ops-history.md`).
